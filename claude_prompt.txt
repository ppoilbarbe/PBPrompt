Generate a complete Python project for a graphical application named PBPrompt using PySide6.
The project description is "A program to register and categorize AI prompts to keep tries."
The publisher name is PBMou.


Required features:

1) What the program does
- The program holds a list of AI prompts to keep.
- There are 5 columns in the list: AI (name of the AI used), Group (a prompt group),
  Name (entry title), Local language (prompt in the local language), English (prompt in English).
- Rows can be selected. Multiple rows can be selected at once.
- The Local language and English column cells are editable in multiline mode:
  a double-click opens an inline editor (QPlainTextEdit) allowing multi-line input.
  Enter confirms the edit and closes the editor (same as Tab). Ctrl+Enter or Shift+Enter
  insert a real line break at the cursor position. Tab / Shift+Tab confirm and move to
  the next / previous cell. Escape cancels; clicking outside confirms.
- Each row height adapts automatically to the text length (including line breaks) so
  that all content is fully visible.
- Long lines that do not fit in width are word-wrapped in the display only
  (without modifying the data).
- Real line breaks in the data are shown in the display with the Unicode character
  ↵ (U+21B5) placed just before the line break.
- A right-click on a cell shows a context menu with Copy / Cut / Paste actions for
  the clipboard. The keyboard shortcuts Ctrl+C / Ctrl+X / Ctrl+V are also available
  from the list. The context menu is translated via the i18n system (gettext).
- There is a filter for each column to search by regular expression and display only
  matching rows. All column filters are cumulative (a row must match all active filters
  to be shown).
- Filters are persistent: their values are saved in the configuration on window close
  (closeEvent) and restored on next startup.
  Implementation: column_filters: dict[str, str] field in AppConfig (keys = column names:
  "ai", "group", "name", "local", "english"); restoration via widget.setText() in
  _connect_signals() (triggers set_filter via textChanged); save in closeEvent() before
  config.save().
- Each filter field (QLineEdit) shows a tooltip stating that regular expressions are
  accepted (e.g. "Filter the AI column. Supports regular expressions.").
  Tooltips are defined in retranslate_ui() and translated via i18n.
- There are buttons in the toolbar:
  - New prompt (to create a new empty row, shortcut Ins),
  - Duplicate (to duplicate the current row and insert it immediately below,
    shortcut Ctrl+D), placed between the New prompt and Delete buttons,
  - Delete (to delete the selected rows after confirmation, shortcut Del),
  - Right arrow to translate from the language in the "Local language" column
    (configured in Settings) to English (shortcut F6),
  - Left arrow to translate from English to the "Local language" column language
    (shortcut F7).
- When one of the two translation buttons is clicked, any in-progress edit is committed
  before the translation runs. Implementation: retrieve the currently focused widget via
  QApplication.focusWidget(); if it is a descendant of the table view
  (tableView.isAncestorOf(fw)), call fw.clearFocus() to unconditionally trigger FocusOut
  → commitData + closeEditor via the delegate's eventFilter. Do NOT use
  tableView.setFocus(): Qt cancels the commit if the target widget is an ancestor of
  the editor.
- The current cell (currentIndex) must look visually different from all other cells in
  the same selected row. Implement this via a dedicated CurrentCellHighlightDelegate
  (QStyledItemDelegate) that:
  * stores a reference to the view (self._view),
  * in paint(), compares the index with self._view.currentIndex() and, if the row is
    selected (State_Selected), draws a 2 px coloured border around the current cell
    (colour: Highlight palette darkened by 160%).
  * Applied to the AI, Group, Name columns via setItemDelegateForColumn().
  * MultiLineDelegate (Local and English columns) inherits from
    CurrentCellHighlightDelegate (not QStyledItemDelegate directly). Its constructor
    takes (view, parent) and calls super().__init__(view, parent). The paint() method
    calls self._draw_current_highlight() at the end of text painting.
- If a translation fails (missing or invalid API key, network error, quota exceeded,
  etc.), an error dialog (QMessageBox.critical) appears with the exact message returned
  by the service. Processing stops at the first error. Rows already translated before
  the error keep their result. The original text is never copied in place of a missing
  translation.
- The primary storage format is SQLite (.sqlite). YAML files are used only for
  import/export (no direct YAML save).
- Each row is an entry in the "prompts" table with columns:
  ai, group, name, local, english (all TEXT NOT NULL DEFAULT '').
  "group" is a SQL keyword and must always be quoted in all SQL queries.
- The SQLite file also contains a "pbprompt" table (signature) with columns
  key TEXT PRIMARY KEY and value TEXT NOT NULL DEFAULT ''. It stores in particular
  key='version' with the version number of the application that wrote the file.
  Full schema:
    CREATE TABLE IF NOT EXISTS pbprompt (key TEXT PRIMARY KEY, value TEXT NOT NULL DEFAULT '')
    CREATE TABLE IF NOT EXISTS prompts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        ai TEXT NOT NULL DEFAULT '', "group" TEXT NOT NULL DEFAULT '',
        name TEXT NOT NULL DEFAULT '', local TEXT NOT NULL DEFAULT '',
        english TEXT NOT NULL DEFAULT '')
  WAL mode enabled (PRAGMA journal_mode=WAL). Row factory = sqlite3.Row.
- Logic for opening a SQLite file (PromptCollection.load):
  * If the "pbprompt" table does not exist → ValueError("Not a PBPrompt database: {path}")
    The UI catches ValueError separately and displays the error message + path
    (could not open file:\n{path}\n\n{error}) then loads an empty collection.
  * If "pbprompt" exists but the "prompts" table is missing columns →
    ALTER TABLE ADD COLUMN "col" TEXT NOT NULL DEFAULT "" for each missing column,
    then update the version. Return the list of added columns.
  * Read all rows with SELECT ai, "group", name, local, english FROM prompts ORDER BY id.
- Saving (PromptCollection.save): creates/updates the SQLite file via
  _create_schema (CREATE TABLE IF NOT EXISTS) + DELETE FROM prompts + INSERT for
  each entry + INSERT OR REPLACE INTO pbprompt key='version'.
- On save (Save As), if the entered filename has no .sqlite extension, it is added
  automatically.
- YAML import (File menu → Import YAML → submenu):
  * "Add entries…": YAML entries are appended to the collection.
  * "Replace all…": the collection is cleared then replaced by the YAML content.
  * Unknown YAML keys (not matching a column) are ignored.
  * Missing keys in the YAML default to "".
  * After import, set_collection() is called to refresh the Qt display.
- YAML export (File menu → Export YAML…):
  * All columns in PROMPTS_COLUMNS are always written, even if empty.
  * The .yaml extension is added automatically if the name does not end in .yaml or .yml.
- The constant PROMPTS_COLUMNS: tuple[str, ...] = ("ai", "group", "name", "local", "english")
  is the single source of truth for column order in SQL, YAML, and the display.
- The recent files history reflects SQLite files (not YAML).
- The File menu contains a "Recent Files" submenu listing recently loaded data files.
  This submenu ends with a separator followed by a "Clear list" entry that clears the
  list. Clicking a file in the list opens it (with unsaved-changes detection). If the
  file no longer exists, a warning is shown, the entry is removed from the list, and an
  empty collection is displayed.
- On application startup, if no file is provided on the command line, the last used file
  is loaded automatically and silently. If it no longer exists, the list is updated
  (entry removed) and an empty collection is displayed.
- The recent files list (full paths, reverse chronological order) is persisted in the
  YAML configuration file.
- Column headers display tooltips describing each column and listing non-standard
  keyboard shortcuts:
  * AI, Group, Name: column description + secondary sort behaviour
    (e.g. "Primary sort on AI → Group → Name").
  * Local language, English: description + multiline editor shortcuts
    (Enter: confirm · Ctrl+Enter or Shift+Enter: line break ·
    Tab / Shift+Tab: next/previous cell · Escape: cancel).
  * IMAGE: description + interaction shortcuts on 2 lines:
    "Double-click to view · Right-click for options\n
    Enter: load from file  ·  Ctrl+V: paste  ·  Backspace: clear"
  Tooltips are translated via the i18n system (gettext).
  Implementation: PromptTableModel.headerData() handles Qt.ToolTipRole via a
  _header_tooltips (list[str]) field and a set_header_tooltips(tooltips: list[str])
  method symmetric to set_header_labels(). Tooltips are set in retranslate_ui() and

Critical points for keyboard shortcuts — Python MRO bug:
  * MainWindow inherits from both QMainWindow and Ui_MainWindow.
    Python's Method Resolution Order (MRO) causes MainWindow.retranslate_ui()
    to SHADOW Ui_MainWindow.retranslate_ui(): when setup_ui() calls self.retranslate_ui(),
    Python dispatches to MainWindow.retranslate_ui() and NEVER calls
    Ui_MainWindow.retranslate_ui().
  * Consequence: all setShortcut() calls in Ui_MainWindow.retranslate_ui() are dead
    code — shortcuts defined in *_ui.py are NEVER installed at runtime.
  * Fix: place ALL setShortcut(QKeySequence(...)) calls directly in
    MainWindow.retranslate_ui() with an explanatory comment:

    # Keyboard shortcuts — set here because Ui_MainWindow.retranslate_ui is
    # shadowed by this override (Python MRO) and therefore never called at runtime.
    self.actionFileNew.setShortcut(QKeySequence("Ctrl+N"))
    self.actionFileOpen.setShortcut(QKeySequence("Ctrl+O"))
    self.actionFileSave.setShortcut(QKeySequence("Ctrl+S"))
    self.actionFileSaveAs.setShortcut(QKeySequence("Ctrl+Shift+S"))
    self.actionFileClose.setShortcut(QKeySequence("Ctrl+W"))
    self.actionFileQuit.setShortcut(QKeySequence("Ctrl+Q"))
    self.actionToolsOptions.setShortcut(QKeySequence("Ctrl+,"))
    self.actionNewPrompt.setShortcut(QKeySequence("Ins"))
    self.actionDuplicatePrompt.setShortcut(QKeySequence("Ctrl+D"))
    self.actionDeletePrompts.setShortcut(QKeySequence("Del"))
    self.actionTranslateToEnglish.setShortcut(QKeySequence("F6"))
    self.actionTranslateFromEnglish.setShortcut(QKeySequence("F7"))
    self.actionImportYamlAdd.setShortcut(QKeySequence("Ctrl+I"))
    self.actionImportYamlReplace.setShortcut(QKeySequence("Ctrl+Shift+I"))
    self.actionExportYaml.setShortcut(QKeySequence("Ctrl+E"))

  * QKeySequence must be imported from PySide6.QtGui.

Critical points for the status bar and action tooltips:
  * Qt automatically shows the statusTip of a QAction in the status bar when the user
    hovers over a menu item OR a toolbar button. (toolTip is reserved for floating
    button tooltips.)
  * To show tooltip + shortcut in the status bar, add the method _update_status_tips()
    to MainWindow and call it at the end of retranslate_ui():

    def _update_status_tips(self) -> None:
        for action in self.findChildren(QAction):
            tip = action.toolTip()
            if not tip:
                continue
            sc = action.shortcut().toString()
            action.setStatusTip(f"{tip}  ({sc})" if sc else tip)

  * findChildren(QAction) returns all QAction children of the window,
    including those in menus and the toolbar.

Critical points for row heights:
  * resizeRowsToContents() must be called AFTER Qt has finalised the widths of Stretch
    columns (Local language and English). If called too early (before the first layout),
    columnWidth() returns an incorrect value → sizeHint() computes absurd heights.
  * Correct implementation in MainWindow:
    - _resize_rows(): calls resizeRowsToContents() only if isVisible() is True;
      does not reschedule via singleShot if the window is not visible.
    - showEvent() (override): calls
      QTimer.singleShot(0, self.tableView.resizeRowsToContents)
      to trigger resizing after the first Qt layout pass (Stretch columns have their
      real width at that point).
    - _on_section_resized(col, old, new): calls resizeRowsToContents() when the LOCAL
      or ENGLISH columns are resized (handles user-driven dynamic resizing).
  * MultiLineDelegate.sizeHint() uses self._view.columnWidth(col) to compute the height
    needed for the text with word-wrap. A fallback of 100 px is applied if the returned
    width is ≤ 8 px (before the first layout).

Critical points for the multiline editor (MultiLineDelegate.eventFilter):
  * Enter / Return alone → commit the edit (commitData + closeEditor).
  * Ctrl+Enter or Shift+Enter → insert a real line break (insertPlainText("\n")).
    Implementation: if event.modifiers() & (Qt.ControlModifier | Qt.ShiftModifier)
  * Tab → commit and move to the next cell (EditNextItem).
  * Backtab (Shift+Tab) → commit and move to the previous cell (EditPreviousItem).
  * Up / Down → natural cursor movement in QPlainTextEdit (return False).
  * Escape / FocusOut → handled by the base class (cancels on Escape).

7) Internationalisation (i18n)
- Tooling: pybabel exclusively (not xgettext/msgmerge/msgfmt/msginit).
  * pot:      pybabel extract -F babel.cfg -k _ --input-dirs=src -o locales/messages.pot
  * merge-po: pybabel update -i locales/messages.pot -d locales -l <lang> -D messages
              then: python tools/fix_po_files.py (normalises headers, removes obsoletes)
  * translations: pybabel compile -f -i <file>.po -o <file>.mo
  babel.cfg at the project root: [python: **.py]

  src/**.py relative to the input-dirs root, giving src/src/**.py (path doubling).

9) RST documentation (Sphinx)
- REQUIRED: validate every modified .rst file with `python3 -m docutils <file> /dev/null`
  before any commit. The command must return exit code 0 with no error messages.
- Never use RST grid tables (``+----+----+``). Use ``.. list-table::`` exclusively:
  this format imposes no column-alignment constraint and avoids "Malformed table" /
  "unexpected indentation" errors caused by cells wider than the defined column width.
- Indentation in directives (``.. note::``, ``.. warning::``, ``.. code-block::``
  etc.): always 3 spaces, never tabs.
- REQUIRED after any .rst file or Python docstring modification: verify with
  `make clean all docs` (clean environment, recompiles everything).
  The command must complete without any Sphinx WARNING or ERROR.

11.1) Automatic Read the Docs version cleanup

After each release, the CI "release" job runs tools/rtd_cleanup.py to deactivate
obsolete versions in the RTD version dropdown.

Retention policy (current version X.Y.Z):
  * Prior major versions (< X):
      Keep only the latest release of the 3 most recent major versions.
  * Current major version X, prior minor versions (< Y):
      For each of the 3 most recent minor versions, keep only the latest patch.
  * Major X, minor Y (current), prior patches (< Z):
      Keep the 3 most recent patches.
  * The pseudo-versions "latest" and "stable" are never touched.

Implementation — tools/rtd_cleanup.py:
  * RTD v3 API calls (GET /versions/?active=true&type=tag, PATCH /versions/{slug}/).
  * Pagination handled ("next" field in the response).
  * SemVer(NamedTuple): (major, minor, patch, slug) — natively sortable.
  * compute_slugs_to_keep(versions) applies the policy and returns the set of slugs
    to keep.
  * Out-of-policy versions are deactivated (active=False), not deleted.
  * CLI parameters: --project (default "pbprompt"), --token (required), --dry-run,
    --verbose.
  * continue-on-error: true in CI (cleanup failure does not block the release).
  * GitHub secret required: READTHEDOCS_TOKEN (RTD API token with write access).

CI integration (.github/workflows/ci.yml):
  * "Clean up old Read the Docs versions" step added at the end of the "release" job.
  * Runs only on semver tags (the "release" job is already conditional).

Critical points for PyInstaller artefacts in CI:
  * make dist produces dist/pbprompt-VERSION-OS-ARCH[.exe], not dist/pbprompt.
    Upload paths must use globs:
      Linux:   path: dist/pbprompt-*-linux-*
      macOS:   path: dist/pbprompt-*-darwin-*
      Windows: path: dist/pbprompt-*.exe
  * upload-artifact@v6 STRIPS the dist/ prefix: files land at the root of the
    artefact (e.g. executable-linux/pbprompt-1.8.0-linux-x86_64, without dist/).
    The "Rename executables" step in the release job must therefore use:
      mv executable-linux/pbprompt-*       dist/pbprompt-${TAG}-linux-amd64
      mv executable-windows/pbprompt-*.exe dist/pbprompt-${TAG}-windows-amd64.exe
      mv executable-macos/pbprompt-*       dist/pbprompt-${TAG}-macos-arm64

Font handling in frozen builds — critical points:
  The libfontconfig.so bundled by PyInstaller has its default config path hardcoded
  to the build machine's conda prefix.  On any other machine that path is absent and
  fontconfig fails silently.  Three-layer solution:

  1. environment.yml declares fonts-conda-ecosystem; pbprompt.spec bundles
     $CONDA_PREFIX/fonts/ as "fonts/" in the binary.

  2. hooks/rthook_fonts.py (runtime hook, Linux only):
     * Writes fonts.conf into sys._MEIPASS with <dir>{_MEIPASS}/fonts</dir> and
       <include ignore_missing="yes">/etc/fonts/fonts.conf</include>.
     * Sets os.environ["FONTCONFIG_FILE"] to that file.
     * Calls ctypes.CDLL("libfontconfig.so.1").FcFini() then .FcInit() to force
       fontconfig reinitialisation before Qt loads.
     * Runs only when sys.frozen is True and fonts/ directory is present.

  3. _load_bundled_fonts(app) in __main__.py, called just after QApplication():
     * Active only when sys.frozen is True.
     * Registers every *.ttf from sys._MEIPASS/fonts/ into QFontDatabase via
       addApplicationFont().
     * Explicitly sets Ubuntu as the application font if "Ubuntu" is in the loaded
       families, bypassing fontconfig for font-family selection.
     * Rationale: fontconfig-based font selection is unreliable in the bundle even
       with the hook (FcFini/FcInit may reinitialise the wrong library instance).
       Forcing Ubuntu guarantees a consistent, known-good UI font on any machine.


14) IMAGE column

The grid has six columns: AI, Group, Name, Image, Local language, English.
(Column enum: AI=0, GROUP=1, NAME=2, IMAGE=3, LOCAL=4, ENGLISH=5)

Database storage:
  * Two BLOB columns in the "prompts" table: image (full image) and thumbnail
    (small image for display in the grid).
  * These columns are nullable, with no DEFAULT (unlike the text columns).
  * SQL migration adds these columns with ALTER TABLE (no DEFAULT for BLOBs).
  * PROMPTS_COLUMNS = ("ai", "group", "name", "local", "english") — text only.
  * PROMPTS_BLOB_COLUMNS = ("image", "thumbnail").

PromptEntry (data.py):
  * New fields: image: bytes | None = None, thumbnail: bytes | None = None
    placed between name and local.
  * to_dict(): encodes the full image in Base64 ASCII if present.
  * from_dict(): decodes Base64 if present.
  * Thumbnails are NOT exported to YAML (only the full image).

Thumbnails:
  * Configurable size in Options (width × height in pixels, default 64×64).
  * Created automatically when an image is added (file load, paste, YAML import).
  * Changing the size in Options does NOT recreate existing thumbnails.
  * "Refresh Thumbnails" action (Tools menu) → recreate all thumbnails in the database.
  * Generation: QImage.scaled(w, h, KeepAspectRatio, SmoothTransformation) → PNG bytes.
  * Format detection: by magic bytes (not by extension):
    - JPEG: data[:3] == b'\xff\xd8\xff'
    - PNG:  data[:8] == b'\x89PNG\r\n\x1a\n'

Module image_utils.py (src/pbprompt/gui/):
  * detect_image_format(data) → "jpeg" | "png" | None
  * generate_thumbnail(image_data, width, height) → bytes | None (PNG)
  * pixmap_from_bytes(data) → QPixmap | None
  * qimage_to_bytes(image, fmt="PNG") → bytes | None
  * ImageViewDialog(QDialog) — full-screen image viewer dialog with scroll.
    - Mouse-wheel zoom: centred on the pixel under the cursor (zoom-to-cursor).
      Algorithm: capture the fraction (fx, fy) of the label under the cursor before
      zooming via label.mapFrom(viewport, pos), then after _apply_mode() reposition
      the scrollbars with hbar.setValue(round(fx*new_w) - cursor_vp.x()).
      QScrollArea::setWidget reparents the label on the viewport → mapFrom is reliable.
      setValue clamps automatically to [min, max] (best possible approximation).

Display and interactions:

  * Double-click → ImageViewDialog (full image).
  * Right-click → image-specific context menu:
    - "Load image from file…": QFileDialog, filter *.jpg *.jpeg *.png
    - "Paste image": image from clipboard OR file path as text
    - (separator)
    - "Copy image": copies the full image to the clipboard (setPixmap)
    - "Save image to file…": QFileDialog Save, filter + suffix from detect_image_format()

    - "Clear image": clears image + thumbnail
